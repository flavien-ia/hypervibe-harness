# /prof

Vous explique en français simple comment Hypervibe fonctionne et à quoi sert chaque brique technique. Avec des analogies du quotidien et autant de questions que vous voulez, pour devenir autonome sans devoir être développeur.

## Quand l'utiliser

- Vous découvrez le plugin et vous voulez comprendre comment il marche **avant** de l'utiliser
- Vous voulez savoir à quoi sert telle ou telle commande (`/add-stripe`, `/add-i18n`…) avant de la lancer
- Vous avez une question sur la stack technique (qu'est-ce que Next.js ? À quoi sert Vercel ?)
- Vous voulez comprendre ce qu'est le "vibe coding", le "déploiement", le "cahier des charges"

## Comment ça se passe

Hypervibe vous propose une **table des matières en 8 sujets** :

1. **Le principe** : Comment ça marche, en 2 minutes
2. **La stack technique** : Les briques de votre app (et à quoi sert chacune)
3. **Le bootstrap** : Ce qui se passe quand vous lancez `/bootstrap`
4. **Les outils du quotidien** : Les commandes que vous utiliserez régulièrement
5. **Les addons** : Les fonctionnalités qu'on ajoute selon vos besoins
6. **Le déploiement** : Comment votre app passe de votre ordinateur à internet
7. **Le cahier des charges** : Comment bien décrire ce qu'on veut construire
8. **Le vibe coding** : Comment faire évoluer votre app après le bootstrap

Vous choisissez le numéro qui vous intéresse, ou vous posez votre propre question en langage libre. Hypervibe répond avec des analogies (restaurant, immeuble, courrier, etc.), explique chaque terme technique au moment où il apparaît, et vous invite à poser plus de questions à la fin.

Si vous dites "tout" ou "commence par le début", elle déroule les 8 sujets dans l'ordre.

## Ce que ça crée pour vous

Rien sur votre ordinateur, c'est une session conversationnelle pure. Mais à la fin, vous comprenez :
- Ce qu'est une "app web" et comment elle est construite
- Ce que font GitHub, Vercel, Neon, et les autres services
- Pourquoi Hypervibe fait tel ou tel choix technique
- Comment vous y prendre pour faire évoluer votre app

## Prérequis

Aucun. `/prof` n'a besoin de rien, vous pouvez la lancer même sans projet existant, juste pour comprendre.

## Astuces

{{callout:tip|Première utilisation d'une nouvelle skill}}
Quand vous découvrez une nouvelle skill du plugin (`/add-stripe`, `/seo`, `/clean`…), lancez `/prof` avant pour demander : "Explique-moi ce que fait `/add-stripe`". Vous lancerez la skill ensuite en sachant à quoi vous attendre.
{{/callout}}

{{callout:info|Vous pouvez aussi poser des questions libres}}
La table des matières est juste un point de départ. Si vous voulez comprendre quelque chose qui n'y figure pas (par exemple "c'est quoi une API ?" ou "pourquoi le RGPD est important ?"), posez la question directement.
{{/callout}}
